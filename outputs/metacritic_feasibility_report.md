# Metacritic Data Enrichment Feasibility Report

## Executive Summary
Enriching the Epic Games Store free games dataset with Metacritic data is **highly feasible**. A preliminary scraping script was able to successfully search for games on Metacritic and extract valuable metadata.

By querying the Metacritic search endpoint (`https://www.metacritic.com/search/{game_name}/`), the site embeds a JSON representation of the search results in the HTML response (within a `<script type="application/json">` block). This allows us to parse the data structure directly without needing to load or execute any JavaScript, bypassing complex web scraping techniques. The data uses a custom reference-based structure (likely generated by Nuxt.js/Vue), but it can be reliably resolved into standard Python objects.

## Data Available from Metacritic
By matching our game titles with Metacritic's search results, we can successfully extract the following data points for each game:

1. **Metacritic Critic Score** (`criticScore`): A highly valuable metric that acts as an industry-standard proxy for "game quality".
2. **Metacritic Title** (`mc_title`): The canonical name of the game as listed on Metacritic.
3. **Metacritic Slug** (`mc_slug`): Useful for constructing direct URLs to the game's Metacritic page or performing future targeted scrapes.
4. **Release Date** (`releaseDate`): The official release date of the game on its primary platform.
5. **ESRB/PEGI Rating** (`rating`): Age ratings like 'E', 'T', 'M', etc.
6. **Platforms** (`platforms`): A comprehensive list of platforms the game is available on (e.g., PC, PlayStation 5, Xbox Series X, Nintendo Switch).

## Technical Implementation Details & Best Practices
- **JSON Payload Parsing:** The search page contains a JSON array. Many objects in this array use integer indices to reference other items in the same array (a technique used to reduce payload size by avoiding data duplication). A recursive resolution function is necessary to reconstruct the full game dictionary.
- **Handling Search Results:** The search can return multiple matches (e.g., movies, other games with similar names). The script filters for items where `type == 'game-title'`, and attempts an exact string match (case-insensitive) against the provided game title. If no exact match is found, it falls back to the first available game result.
- **Rate Limiting:** Metacritic (behind Cloudflare) may block aggressive scraping. Best practices require:
  - Setting a realistic `User-Agent` header.
  - Adding random sleep intervals (e.g., `time.sleep(random.uniform(2, 4))`) between requests.
  - Batching processing or caching results to avoid hitting the endpoints repeatedly for the same games.
- **Cloudflare Protections:** Currently, basic `urllib` requests are succeeding without triggering Cloudflare blocks, likely because the search endpoint does not require complex cookie handling for standard GET requests.

## Proof of Concept
A proof of concept script was successfully executed against a sample of 10 games from the `epic_free_games_with_api_details.csv` dataset. The results are stored in `outputs/metacritic_sample.csv`.

For example, for **"Limbo"**:
- **Critic Score:** 90
- **Rating:** T
- **Platforms:** Xbox 360, PlayStation 3, PC, PlayStation Vita, iOS, Xbox One, PlayStation 4, Nintendo Switch

And for **"Sifu"**:
- **Critic Score:** 81
- **Rating:** M

## Conclusion
Adding Metacritic data will significantly enhance the dataset, allowing for new types of analysis and visualizations (e.g., "Average Critic Score of Free Games over Time", "Are Free Games getting better or worse?"). Given the structured nature of the embedded JSON data, building a robust enrichment pipeline is straightforward and recommended.

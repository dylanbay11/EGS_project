# 2026-04 Update: EGS Project State

## Project Overview
The EGS (Epic Games Store) Project aims to analyze details from Epic Games Store ongoing giveaways, ultimately visualizing this data using a Streamlit application. This report outlines the current state of the codebase, data assets, and identified issues as of April 2026.

## Current State of Development (`development/`)
The `development` directory contains scripts actively used or recently worked on:
- **`data_collection.py`**: A python script that loads the list of free games from a Google Sheet and queries the `epicstore_api` to get more details (like tags, prices). Currently, the API calls fail with decoding errors. (A bug involving an undefined variable `game_title_col_name` was fixed in a recent atomic commit).
- **`scrape_sheet.py`**: A Marimo notebook/script that performs a similar function as `data_collection.py`, fetching the game list from the Google Sheet and detailed info via `epicstore_api`. This script requires the `marimo` package, which was missing from `requirements.txt`.
- **`old_app.py`**: A Streamlit application intended to visualize the data, displaying total dollar value over time, game types, and a game search interface.
- **`old_reviewfetcher.py` / `old_scraper.ipynb`**: Earlier iterations of data retrieval logic.

## Data Assets (`data/`)
The `data` directory holds various datasets used for or generated by the scripts:
- `jan_2_2024_excelexport.xlsx` / `jan_2_2024_sheets_download.csv`: Data dumps of giveaways up to January 2024.
- `csvtry.xlsx`, `tempnumbers.numbers`: Various format experiments.

## Archived Code (`archive/`)
The `archive` directory contains older, largely deprecated scripts and notebooks:
- Numerous Jupyter notebooks detailing the progression of scraping and API querying: `scraper_wip.ipynb`, `working_scraper.ipynb`, `scraper_next.ipynb`, `second_scraper.ipynb`.
- R scripts (`missings.R`) for missing data analysis.
- Previous standalone python scripts (`EGS.py`, `final_app.py`).

## Dependencies and Issues
- The `requirements.txt` file specifies standard data and web tools (`streamlit`, `pandas`, `plotly`, `numpy`, `epicstore_api`). However, it fails to include `marimo`, which is required to run `scrape_sheet.py`.
- **API Issue:** Both `data_collection.py` and `scrape_sheet.py` rely on `epicstore_api`. Currently, the `fetch_store_games` and `get_product` calls are failing with a `json.decoder.JSONDecodeError` (`Expecting value: line 1 column 1 (char 0)`). This suggests the Epic Games Store API structure or authentication has changed and the wrapper is returning invalid/empty responses, effectively halting new data collection.
